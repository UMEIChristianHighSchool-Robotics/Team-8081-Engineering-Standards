# Build Season Timeline

This document defines our structured approach to build season.

Build season success depends on:
- Early strategy alignment
- Parallel development
- Controlled iteration
- Clear deadlines

We follow a phased approach.

---

# 🗓 Phase 0 – Preseason (September–December)

Objectives:
- Train new members
- Practice Git workflow
- Build programming templates
- Practice CAD standards
- Review prior season lessons

Deliverables:
- Updated engineering documentation
- Tested robot template project
- Electrical standards reviewed

---

# 🚀 Phase 1 – Kickoff & Strategy (Week 1)

Objectives:
- Analyze game manual
- Identify scoring priorities
- Define robot archetype
- Establish design constraints

Deliverables:
- Strategy document
- Initial robot concept sketch
- Subsystem list
- Risk assessment

No CAD or coding before strategy alignment.

---

# 🏗 Phase 2 – Prototyping (Weeks 1–2)

Objectives:
- Build fast prototypes
- Test mechanisms physically
- Validate scoring methods
- Identify failure modes

Deliverables:
- Prototype performance data
- Decision on final mechanism direction

Programming:
- Begin updating robot template
- Set up subsystem structure

---

# 📐 Phase 3 – CAD & Architecture (Weeks 2–4)

Objectives:
- Finalize drivetrain design
- Finalize scoring mechanism CAD
- Define control system layout
- Freeze major design decisions

Deliverables:
- Full robot CAD
- Electrical layout plan
- Confirmed CAN ID map

Programming:
- Implement subsystem stubs
- Simulate commands
- Begin drive testing

---

# 🔩 Phase 4 – Fabrication & Integration (Weeks 4–6)

Objectives:
- Manufacture parts
- Assemble robot
- Wire electronics
- Deploy first working code

Deliverables:
- Drivetrain operational
- Basic scoring mechanism working
- Baseline autonomous functional

Priority:
Drive > Score Preload > Add Complexity

---

# 🧪 Phase 5 – Testing & Iteration (Weeks 6–8)

Objectives:
- Tune PID
- Test autonomous routines
- Practice driver control
- Improve reliability

Deliverables:
- ≥90% autonomous success rate
- No electrical reliability issues
- Version-tagged competition build

No major redesign after this phase.

---

# 🏁 Phase 6 – Competition Readiness

Checklist:
- Robot weight confirmed
- All fasteners secured
- Wiring inspected
- Code tagged and stable
- Backup batteries ready
- Spare parts prepared

Stability > Last-minute improvements

---

# 🧠 Build Season Rules

- Strategy drives design.
- Prototype before committing.
- No massive refactors near competition.
- Documentation updated weekly.
- Lessons learned recorded after each event.

We build with intention, not panic.
